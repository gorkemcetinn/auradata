## What is AuraData?

AuraData lets you drop a CSV or Excel file and immediately receive:

- рҹ“Ҡ **Smart chart recommendations** вҖ” bar, line, pie, and scatter вҖ” with the right column mappings picked automatically
- рҹ“қ **Editorial narrative** вҖ” a human-sounding analysis of your data generated by an LLM
- рҹҺЁ **Interactive canvas** вҖ” drag, resize, and arrange charts on a collaborative dashboard
- рҹҢҗ **i18n support** вҖ” Turkish / English toggle built in
- рҹҢҷ **Dark & light mode** вҖ” persisted via `next-themes`

The entire analysis pipeline lives in an **n8n workflow** вҖ” no custom backend server required.

## Architecture

```
Browser (Next.js on Vercel)
       в”Ӯ
       в”Ӯ  POST /webhook/analyze  (JSON)
       в–ј
  в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
  в”Ӯ        n8n  (self-hosted VPS)       в”Ӯ
  в”Ӯ                                     в”Ӯ
  в”Ӯ  Webhook вҶ’ stats-helper вҶ’ LLM API  в”Ӯ
  в”Ӯ              в”Ӯ                      в”Ӯ
  в”Ӯ       Flask + pandas                в”Ӯ
  в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
       в”Ӯ
       в”Ӯ  Supabase (Auth + analyses table)
       в–ј
   PostgreSQL
```

1. **Frontend** sends the parsed dataset (first 50 rows as JSON) to the n8n webhook.
2. **n8n** passes the data to the `stats-helper` Flask microservice for column-type inference and statistical summaries.
3. **n8n** forwards the stats summary to a Groq / Gemini LLM and receives structured `{ recommendations, summary }` JSON.
4. **Frontend** renders the charts and narrative; the result is saved to Supabase.

## Getting Started

### Prerequisites

- Node.js вүҘ 20
- Docker & Docker Compose (for n8n + stats-helper)
- A [Supabase](https://supabase.com) project
- A [Groq](https://console.groq.com) API key (free tier available)

---

### 1 вҖ” Clone & install

```bash
git clone https://github.com/gorkemcetinn/auradata.git
cd auradata
npm install
```

---

### 2 вҖ” Set up environment variables

#### Frontend (`.env.local`)

```bash
cp .env.local.example .env.local
```

| Variable | Description |
|---|---|
| `NEXT_PUBLIC_ANALYZE_WEBHOOK_URL` | Full URL of your n8n webhook |
| `NEXT_PUBLIC_USE_MOCK_ANALYZE` | Set `true` to skip n8n in local dev |
| `NEXT_PUBLIC_SUPABASE_URL` | Your Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Your Supabase anon key |

#### Infrastructure (`infra/.env`)

```bash
cp infra/.env.example infra/.env
```

| Variable | Description |
|---|---|
| `N8N_BASIC_AUTH_USER` | n8n login username |
| `N8N_BASIC_AUTH_PASSWORD` | n8n login password |
| `N8N_ENCRYPTION_KEY` | 32-byte hex key вҖ” run `openssl rand -hex 32` |
| `WEBHOOK_URL` | Public URL of n8n (e.g. `http://localhost:5678`) |
| `GROQ_API_KEY` | Groq API key |
| `GEMINI_API_KEY` | Google Gemini API key (optional fallback) |

---

### 3 вҖ” Start the backend services (n8n + stats-helper)

```bash
docker compose -f infra/docker-compose.yml up -d
```

n8n will be available at [http://localhost:5678](http://localhost:5678).

---

### 4 вҖ” Configure n8n workflow

1. Open [http://localhost:5678](http://localhost:5678) and log in.
2. Create a new workflow with the following nodes:
   - **Webhook** вҶ’ path: `analyze`, method: `POST`
   - **HTTP Request** вҶ’ `POST http://stats-helper:5000/analyze` (sends data JSON)
   - **LLM node (Groq or OpenAI)** вҶ’ generates `recommendations` + `summary`
   - **Respond to Webhook** вҶ’ returns the structured JSON
3. Activate the workflow.

> See the [Webhook Contract](#webhook-contract) section below for the exact request / response shape.

---

### 5 вҖ” Run the frontend

```bash
npm run dev
```

Visit [http://localhost:3000](http://localhost:3000).

## Webhook Contract

### Request

```
POST <WEBHOOK_URL>/webhook/analyze
Content-Type: application/json
```

```json
{
  "dataset_name": "sales_2024.csv",
  "data": [{ "month": "Jan", "revenue": 12000 }, "...up to 50 rows"]
}
```

### Response

```json
{
  "summary": "The dataset shows a strong upward trend in Q3...",
  "recommendations": [
    {
      "id": "rec-1",
      "type": "bar",
      "x_column": "month",
      "y_column": "revenue",
      "title": "Monthly Revenue",
      "description": "Best for comparing discrete categories."
    }
  ]
}
```

## stats-helper Microservice

A minimal Flask service that receives the raw data and returns structured column metadata for the LLM prompt.

**Endpoint:** `POST http://stats-helper:5000/analyze`  
**Content-Type:** `application/json`

```json
{
  "columns": [
    { "name": "month", "dtype": "categorical" },
    { "name": "revenue", "dtype": "numeric" }
  ],
  "stats": {
    "revenue": { "min": 5000.0, "max": 95000.0, "mean": 42300.0, "std": 18200.0 }
  },
  "sample_rows": [{ "month": "Jan", "revenue": 12000 }]
}
```

## Deployment

### Frontend вҶ’ Vercel (free)

```bash
npm i -g vercel
vercel --prod
```

Set all `NEXT_PUBLIC_*` variables in the Vercel dashboard under **Settings вҶ’ Environment Variables**.

### n8n + stats-helper вҶ’ VPS

Any вӮ¬4вҖ“6/month VPS (Hetzner CX11, DigitalOcean Droplet) or **Oracle Cloud Always Free** works.

```bash
# On the VPS
curl -fsSL https://get.docker.com | sh
git clone https://github.com/your-username/auradata.git
cd auradata
cp infra/.env.example infra/.env  # fill in your values
docker compose -f infra/docker-compose.yml up -d
```

Use **Caddy** for automatic HTTPS (no certbot needed):

```caddyfile
# /etc/caddy/Caddyfile
n8n.yourdomain.com {
    reverse_proxy localhost:5678
}
```

Update `WEBHOOK_URL` in `infra/.env` and `NEXT_PUBLIC_ANALYZE_WEBHOOK_URL` in Vercel to your live domain.
