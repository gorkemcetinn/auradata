## What is AuraData?

AuraData lets you drop a CSV or Excel file and immediately receive:

- 📊 **Smart chart recommendations** — bar, line, pie, and scatter — with the right column mappings picked automatically
- 📝 **Editorial narrative** — a human-sounding analysis of your data generated by an LLM
- 🎨 **Interactive canvas** — drag, resize, and arrange charts on a collaborative dashboard
- 💬 **AI Chat & Canvas Chat** — talk directly to your data and generate new insights interactively
- 🌐 **i18n support** — Turkish / English toggle built in
- 🌙 **Dark & light mode** — persisted via `next-themes`

The entire analysis pipeline lives in an **n8n workflow** on Oracle Cloud — no custom backend server required.

## Architecture

```
Browser (Next.js on Vercel)
       │
       │  POST /webhook/analyze
       │  POST /webhook/chat
       │  POST /webhook/canvas-chat
       ▼
  ┌─────────────────────────────────────┐
  │        n8n  (Oracle Cloud VM)       │
  │                                     │
  │  Webhook Nodes → LLM Processing     │
  │  (Data parsing, prompt assembly)    │
  └─────────────────────────────────────┘
       │
       │  Supabase Client (Browser)
       ▼
   PostgreSQL (Supabase)
```

1. **Frontend** sends data/prompts via REST to the live n8n webhook URLs.
2. **n8n** receives the webhook, orchestrates the LLM API calls, structures the AI's response, and returns it to the frontend.
3. **Frontend** renders the UI (Charts, Narrative, Chat responses) and persists the state directly to **Supabase**.

## Webhook Contracts

AuraData uses three primary webhooks exposed by n8n. The frontend environment variables `NEXT_PUBLIC_*_WEBHOOK_URL` map to these endpoints.

### 1. Analysis Webhook (`/webhook/analyze`)
Handles the initial dataset upload and returns chart recommendations and a narrative summary.

**Request** (`POST`)
```json
{
  "dataset_name": "sales_2024.csv",
  "data": [{ "month": "Jan", "revenue": 12000 }, "...up to 50 rows"]
}
```

**Response** (`JSON`)
```json
{
  "summary": "The dataset shows a strong upward trend in Q3...",
  "recommendations": [
    {
      "id": "rec-1",
      "type": "bar",
      "x_column": "month",
      "y_column": "revenue",
      "title": "Monthly Revenue",
      "description": "Best for comparing discrete categories."
    }
  ]
}
```

### 2. General Chat Webhook (`/webhook/chat`)
Handles the floating AI assistant chat queries.

**Request** (`POST`)
```json
{
  "message": "En çok satış yapılan ay hangisi?",
  "context": "Contextual data from the current page if any"
}
```

**Response** (`JSON`)
```json
{
  "reply": "Verilerinize göre en çok satış Kasım ayında (120,000₺) yapılmıştır."
}
```

### 3. Canvas Chat Webhook (`/webhook/canvas-chat`)
Handles deep-dive questions specific to the interactive canvas dashboard.

**Request** (`POST`)
```json
{
  "query": "Bu grafikteki aykırı değerleri açıkla",
  "chartData": { ... },
  "canvasState": { ... }
}
```

**Response** (`JSON`)
```json
{
  "reply": "Grafikte görülen aykırı değerler yaz aylarındaki promosyonlardan kaynaklanmaktadır..."
}
```

## Deployment Environment

- **Frontend:** Hosted on Vercel. Connects to Supabase via `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY`.
- **Backend (n8n):** Hosted on an Oracle Cloud `VM.Standard.E2.1.Micro` instance running Ubuntu 24.04.
  - Deployed via Docker Compose.
  - Port 5678 is open via iptables and Oracle Security Lists.
  - Operates 24/7 without needing an active SSH session.
